### **CapCut 推荐首页重构与性能优化 (2024.04 - 2025.03)**

#### **背景**

CapCut“剪同款”推荐首页作为**日均 PV 达 1.7 亿**的核心流量入口，历经两年高速迭代后，原有技术架构已无法支撑业务发展。代码（**~7 万行**）高度耦合、启动流程混乱、性能问题突出（**Android FMP P90 高达 15.5s**），严重制约了开发效率与用户体验。

#### 架构设计与重构

- **UI/逻辑彻底分离**：将项目重构为 **UI 层、逻辑层、领域层**三层架构，彻底解耦视图与业务逻辑。
- **领域模型封装**：引入 **DDD（领域驱动设计）** 思想，将缓存管理、实验配置、数据请求、异常处理等通用能力下沉至**领域服务**，解决了底层能力重复建设与管理混乱的问题。
- **依赖注入 (DI)**：通过依赖注入容器管理模块依赖，简化了模块间的复杂调用关系，提升了代码的可测试性与可维护性。
- **启动流程治理**：建立**页面级与容器级生命周期管控**机制，将启动过程拆分为 Open, Prepare, TabReady, Completed, Idle 等多个阶段，实现高优任务前置、低优任务闲时执行，解决了启动逻辑混乱、关键路径被阻塞的问题。
- **FeedList 核心组件重构**：针对 **5k+ 行**的 FeedList 组件，采用**策略模式**与**责任链模式**，将 **20+** 种数据请求场景抽象为独立的 **Handler 节点**与 **Feed 操作策略**，大幅降低了组件的圈复杂度。

#### 性能优化策略

- **数据预请求 (Prefetch)**：与客户端协同，将核心 FeedList 数据的网络请求**提前至 App 冷启动阶段**，前端通过 JSB 直接获取预请求结果，实现接口“零等待”。
- **首屏可见渲染优先**：通过**曝光渲染机制**与业务逻辑判断，精准渲染首屏可见节点，剔除无关节点的初始渲染，显著减少了首次渲染的计算量。
- **Code Cache 优化**：接入 **Lynx Code Cache** 方案，缓存 JS 代码解析后的结果，将二次加载时的**业务 JS 加载与执行耗时降低**，对安卓低端机提升尤为明显。
- **服务端协同优化**：推动服务端实施接口数据精简，首屏仅返回必要字段；并探索**图片 Base64 直出**等方案，减少网络 RTT 带来的耗时。
- **防劣化机制建设**：搭建性能监控看板与自动化告警体系，建立双周性能 Review 机制，确保优化成果的长期稳定。

#### 成果

- **性能显著提升：**Android 端 FMP: P90 从 15.5s 优化至 10.5s (-32.2%), P50 从 4.6s 优化至 3.7s；iOS 端 FMP: P90 从 7.0s 优化至 4.4s (-37.1%), P50 从 2.7s 优化至 1.8s。
- **业务收益显著**: 推荐导出渗透率累计正向显著 +0.53% (Android) / +0.15% (iOS)；人均推荐导出次数累计正向显著 +0.75% (Android) / +0.32% (iOS)。
- **研发效率与质量显著提升：**新需求开发效率提升约 30% (周期内对比)；缺陷估分降低约 40%。
